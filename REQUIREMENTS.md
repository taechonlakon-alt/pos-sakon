# POS & Inventory Management System Requirements

## 1. Problem Statement (ปัญหา)
ร้านขายอะไหล่รถจักรยานยนต์ เครื่องมือการเกษตร และเครื่องตัดหญ้า มีสินค้าหลากหลายประเภทและจำนวนมาก ทำให้เกิดปัญหา:
- ไม่ทราบยอดขายรายวันอย่างชัดเจน
- ไม่ทราบกำไรที่แท้จริง
- ไม่ทราบจำนวนสินค้าคงเหลือแบบ Real-time
- ไม่รู้ว่าสินค้าใดขายดี
- ไม่รู้ว่าสินค้าใดค้างสต็อกนาน
- การจดบันทึกแบบกระดาษทำให้ผิดพลาดง่าย
- มีคอมพิวเตอร์หน้าร้านเพียงเครื่องเดียว และลูกค้าอาจเห็นข้อมูลต้นทุน

## 2. Objectives (วัตถุประสงค์ของระบบ)
- พัฒนาระบบบริหารจัดการสินค้าและการขายแบบออนไลน์ (Cloud)
- แยกโหมดใช้งานระหว่างหน้าร้าน (POS) และผู้ดูแล (Admin)
- แสดงยอดขายและกำไรแบบ Real-time
- ควบคุมสิทธิ์การเข้าถึงข้อมูลต้นทุน
- รองรับการดูรายงานผ่านมือถือ

## 3. Functional Requirements (REQ)

### 3.1 ระบบผู้ใช้งาน
- **FR-01** ระบบต้องรองรับการเข้าสู่ระบบ (Login)
- **FR-02** ระบบต้องกำหนดสิทธิ์ผู้ใช้งานเป็น Admin และ Staff
- **FR-03** Staff ต้องไม่สามารถดูต้นทุนและกำไรได้
- **FR-04** Admin ต้องสามารถดูรายงานทั้งหมดได้

### 3.2 ระบบจัดการสินค้า
- **FR-05** เพิ่มสินค้าได้
- **FR-06** แก้ไขสินค้าได้
- **FR-07** ลบสินค้าได้
- **FR-08** กำหนดหมวดหมู่สินค้าได้
- **FR-09** กำหนด SKU ได้
- **FR-10** กำหนดราคาขายได้
- **FR-11** บันทึกต้นทุนแบบเป็นล็อตได้
- **FR-12** แจ้งเตือนสินค้าใกล้หมดได้

### 3.3 ระบบขายหน้าร้าน (POS)
- **FR-13** ค้นหาสินค้าด้วยชื่อหรือ SKU ได้
- **FR-14** เพิ่มสินค้าลงตะกร้าได้
- **FR-15** แก้ไขจำนวนสินค้าในตะกร้าได้
- **FR-16** คำนวณยอดรวมอัตโนมัติ
- **FR-17** บันทึกการขายได้
- **FR-18** ตัดสต็อกอัตโนมัติ
- **FR-19** คำนวณกำไรตามต้นทุนจริง (FIFO)

### 3.4 ระบบรายงาน
- **FR-20** แสดงยอดขายรายวัน
- **FR-21** แสดงยอดขายรายเดือน
- **FR-22** แสดงกำไรรวม
- **FR-23** แสดงสินค้าขายดี
- **FR-24** แสดงสินค้านอนสต็อก
- **FR-25** Export รายงานเป็น CSV ได้

### 3.5 การเข้าถึงผ่านมือถือ
- **FR-26** Admin ต้องสามารถดู Dashboard ผ่านมือถือได้
- **FR-27** ข้อมูลต้องอัปเดตแบบ Real-time

## 4. Non-Functional Requirements
### Security
- ใช้ Role-Based Access Control
- ซ่อนต้นทุนจากหน้า POS
- ใช้ HTTPS
- Auto Logout เมื่อไม่ใช้งาน

### Performance
- โหลดหน้าไม่เกิน 3 วินาที
- รองรับข้อมูลสินค้าอย่างน้อย 5,000 รายการ

### Availability
- ระบบต้องออนไลน์ 99% ต่อเดือน
- มีระบบ Backup ข้อมูล

### Usability
- หน้า POS ต้องใช้งานง่าย
- ปุ่มขนาดใหญ่
- ใช้งานได้ในหน้าจอสัมผัส

## 5. Constraints
- มีคอมพิวเตอร์หน้าร้านเพียง 1 เครื่อง
- ต้องใช้ระบบ Cloud
- งบประมาณจำกัด (ใช้ Free tier)

## Assumptions
- ร้านมีอินเทอร์เน็ตตลอดเวลา
- เจ้าของร้านใช้งานมือถือดูรายงาน
- มีผู้ใช้งานไม่เกิน 5 คน
